# 临时工作提醒网页部署包

这个文件夹可以直接部署成一个公网网址，办公室电脑、家里电脑、手机浏览器都能打开。

## 最快上线方式

推荐使用 Netlify Drop、Vercel、Cloudflare Pages 或 GitHub Pages 这类静态网站托管服务。

如果你想今晚最快拿到网址：

1. 打开 Netlify Drop。
2. 把整个 `reminder-web-deploy` 文件夹拖进去。
3. 等它生成一个 `https://...netlify.app` 网址。
4. 明天在办公室电脑打开这个网址即可。

## 很重要：数据是否同步

当前版本的数据保存在浏览器本地。

这意味着：

- 同一个网址可以在不同设备打开。
- 但每台设备里的提醒记录默认不互通。
- 如果你在家里电脑录了 5 条，办公室电脑第一次打开不会自动看到这 5 条。

临时解决方式：

- 在家里电脑点击“导出”。
- 到办公室电脑打开同一个网址后点击“导入”。

真正同步方式：

- 增加登录能力。
- 把提醒数据保存到云数据库。
- 常见选择：Supabase、Firebase、Cloudflare D1、LeanCloud 或自建后端。

## 后续更新网页怎么同步

有两种轻便方式：

### 方式一：继续拖拽发布

适合偶尔改页面设计、改小功能。

1. 修改 `index.html`。
2. 重新把 `reminder-web-deploy` 文件夹拖到 Netlify Drop。
3. Netlify 会生成一个新的网址。

缺点：网址可能变化。如果要固定网址，需要登录 Netlify，把新版本部署到同一个站点里。

### 方式二：GitHub + Netlify 自动发布

适合后续经常更新，推荐。

1. 建一个 GitHub 仓库。
2. 把 `index.html` 提交到仓库。
3. 在 Netlify 里选择“Add new site”并连接这个仓库。
4. 以后每次改完页面，只要提交到 GitHub，Netlify 会自动更新同一个网址。

这样页面设计、功能改动都能同步到固定网址，不需要每次手动拖文件。

## 到点提醒说明

当前版本已加入浏览器提醒：

- 点击页面右上角“开启提醒”。
- 浏览器允许通知后，页面保持打开时，到点会弹出通知。
- 如果没有允许通知，页面打开时会用普通弹窗提醒。
- 只有设置了“日期 + 时间”的提醒会触发到点弹出。

限制：

- 纯静态网页无法保证页面关闭后还准时提醒。
- 手机浏览器、电脑浏览器对后台通知限制不同。
- 如果需要“页面关掉也能到点提醒”，需要后端或云函数定时任务，加浏览器 Push 或邮件/短信/企业微信等推送。
